This week I learned about testing and debugging inside the Javascript Language. There are different types of errors in programming (a) Syntax Errors, (b) Runtime Errors, and (c) Logical Errors.

  1. Syntax Errors - parsing errors, occur at compile time in traditional programming languages and at interpret time in JavaScript.
  2. Runtime Errors - errors that cause a program to terminate abnormally
  3. Logic Errors - occur when a program does not perform the way it was intended to

It's important to be able to recognize what errors we have so that we can properly test and debug them.

The try...catch...finally Statement

The try block must be followed by either exactly one catch block or one finally block (or one of both). When an exception occurs in the try block, the exception is placed in e and the catch block is executed. The optional finally block executes unconditionally after try/catch.

Console.log()

One debugging method is console.log(). Using this allows us to display a value in the console so we can see what value is being claculated and can see if it's our desired outcome.

a = 1;
b = 2;
c = a + b;
console.log(c);
//3

By doing this we will see that c is equal to c and and know whether or our program is working correctly or not.

Breakpoints

Another method is by setting breakpoints. We can set breakpoints and when our code runs into one, it will stop executing and we can see the current values. We can then resume the program.

Debugger Keyword

The debugger keyword is similar to settting a breakpoint. It stops the execution of the program where it is placed before continuing.

var x = 15 * 5;
debugger;
document.getElementById("demo").innerHTML = x;
